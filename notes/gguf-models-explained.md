## 技术术语解释

### 模型架构参数

#### block_count (层数)

- **定义**: Transformer 层的数量，即模型的深度
- **影响**: 层数越多，模型理解能力越深，但推理速度越慢
- **类比**: 像一本书的章节数，章节越多，内容越深入

| 模型 | 层数 | 说明 |
| --- | --- | --- |
| 14B | 48层 | 深度理解 |
| 8B | 32层 | 平衡选择 |
| 7B | 67层 | 扩展架构 |
| 1.5B | 28层 | 轻量级 |
| 1B | 16层 | 超轻量 |

#### embedding_length (嵌入维度)

- **定义**: 词嵌入向量的维度，即模型的"宽度"
- **影响**: 维度越大，模型能捕捉的语言特征越丰富
- **类比**: 像画笔的精细度，维度越高，描绘越细腻

| 模型 | 嵌入维度 | 说明 |
| --- | --- | --- |
| 14B | 5120 | 非常精细 |
| 8B | 4096 | 标准配置 |
| 1.5B | 1536 | 轻量级 |
| 1B | 2048 | 基础配置 |

#### feed_forward_length (FFN维度)

- **定义**: 前馈神经网络中间层的维度
- **作用**: 负责模式学习和特征转换
- **通常关系**: FFN维度 ≈ 2.5x ~ 4x 嵌入维度

#### attention.head_count (注意力头数)

- **定义**: 多头注意力机制中的头数
- **作用**: 多个头并行处理不同的语言关系
- **类比**: 像多个侦探同时调查案件的不同方面

#### attention.head_count_kv (KV头数)

- **定义**: Key-Value 头的数量（GQA - 分组查询注意力）
- **优势**: 比标准多头注意力更省内存
- **原理**: 多个 Query 头共享同一组 Key-Value 头

#### context_length (上下文长度)

- **定义**: 模型能处理的最大序列长度
- **影响**: 决定了单次对话能处理多少文本
- **单位**: token（词元）

| 上下文长度 | 约等于 |
| --- | --- |
| 32K | ~2.4万汉字 |
| 128K | ~9.6万汉字 |

#### rope.freq_base (RoPE频率基数)

- **定义**: 旋转位置编码的基础频率
- **作用**: 控制位置信息的编码方式
- **RoPE**: Rotary Position Embedding，旋转位置编码

#### rope.scaling (RoPE缩放)

- **定义**: 扩展上下文长度的技术
- **类型**:
  - `linear`: 线性缩放
  - `dynamic`: 动态缩放
  - `yarn`: Yet Another RoPE extensioN
- **作用**: 允许处理比训练时更长的文本

#### vocab_size (词表大小)

- **定义**: 词表中的 token 数量
- **影响**: 词表越大，多语言支持越好
- **示例**: Qwen2 约15万，Llama 3.1 约12.8万

## 量化类型说明

量化是将模型权重从高精度（如32位浮点）转换为低精度（如4位整数）的过程，以减小模型大小和提高推理速度。

### 量化类型对比

| 类型 | 每权重位数 | 8B模型大小 | 质量 | 适用场景 |
| --- | --- | --- | --- | --- |
| F16 | 16位 | ~16GB | 最高 | 追求极致质量 |
| Q8_0 | 8位 | ~8GB | 很高 | 质量优先 |
| Q6_K | 6位 | ~6GB | 高 | 平衡选择 |
| Q4_K_M | 4位 | ~4GB | 良好 | **推荐默认** |
| Q3_K_S | 3位 | ~3.5GB | 可接受 | 显存紧张 |
| Q2_K | 2位 | ~3GB | 有损失 | 极致压缩 |

### 量化命名规则

- **Q**: Quantized（量化）
- **数字**: 每个权重占用的位数
- **K**: K-quant（K量化，一种优化的量化方法）
- **后缀**:
  - `S`: Small（小，更激进的量化）
  - `M`: Medium（中，平衡选择）
  - `L`: Large（大，更保守的量化）

### 选择建议

```text
显存充足 (>=8GB)  → Q8_0 或 Q6_K
显存中等 (4-8GB)  → Q4_K_M（推荐）
显存紧张 (2-4GB)  → Q3_K_S
极致压缩 (<2GB)   → Q2_K（质量有损）
```

### 为什么 KV头数 < 注意力头数？

这是 **GQA (Grouped Query Attention)** 技术：

```text
标准多头注意力: 32 Query + 32 Key + 32 Value = 96 组
GQA:            32 Query + 8 Key + 8 Value = 48 组
```

**优势**:

- 减少约 50% 的 KV 缓存内存
- 推理速度更快
- 质量损失很小

### 参数量与性能关系

```text
1B-3B  → 快速原型、移动设备、边缘计算
7B-8B  → 通用任务、个人使用、平衡选择
14B+   → 专业任务、高质量输出、需要更多资源
```
